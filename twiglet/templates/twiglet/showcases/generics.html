{% extends "twiglet/showcases/base_showcase.html" %} {% block showcasedescription %} This is a sample program written by Peter Sestoft to show off the use of generic classes in C#. It will convert a regular expression to a deterministic finite automaton (and visualize that DFA by producing a dot file that can be viewed with graphviz).

A fuller description, the source code, and some more nice examples can be found on Peter's web site. The translation from C# to Java shown here is compeletely automatic:

  • Notice how the single C# file is rewritten to 11 Java files, one per class.
  • Normally, we translate references to "built-in" C# types such as int and bool to the corresponding primitive type in Java. However, in some contexts (such as generic type arguments), it is neccessary to translate to the corresponding boxed types. See the translations for int on lines 23 and 24 of Dfa.java.
  • Set<T> implements the ICollection<T> interface. In Set.java you can see that CS2J has added implementations for iterator(), add(), addAll(), clear(), contains(), containsAll(), isEmpty(), remove(), removeAll(), retainAll(), size() and toArray() so that it can also implement the Java interface Collection.
There are many more interesting translations that you can find by studying this example. You can download the code by right clicking on the top most folder. Both are runnable code.
