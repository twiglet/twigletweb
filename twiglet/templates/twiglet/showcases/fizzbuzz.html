{% extends "twiglet/showcases/base_showcase.html" %} {% block showcasedescription %} This is Mike Minutillo's Enterprise worthy implementation of the infamous Fizzbuzz programming challenge. Mike has blogged about his motivation and provided the source code. It provides a nice, well-written C# program to show off some of CS2J's goodies. The translation from C# to Java shown here is compeletely automatic:

  • As is required for Java programs, classes are placed in their own file, in directories named after their package. Packages are named after the namespaces used in the C# program. Minimal, sufficient import statements are automatically inserted into each class definition.
  • All comments are transferred from the C# program into the Java code. In addition, C# documentation comments are translated into Javadoc comments. You can compare the C# documentation to the Java documentation.
  • CS2J uses flexible XML translation templates to guide its translation of external calls. The provided translations for the System.Int32 class guide the translation of number.ToString(CultureInfo.InvariantCulture) (IdentityTransformer.cs, line 34) to NumberFormat.getInstance(LocaleSupport.INVARIANT).format(number) (IdentityTransformer.java, line 35) and make sure the correct imports are added to the Java class.
  • Getter and Setter methods are automatically created for C# properties. References to properties are automatically converted to the appropriate setter/getter call. See the translation for the Message property and its usage from StaticMessageTransformer.cs in StaticMessageTransformer.java.
  • Delegates are fully supported. In FizzBuzz.BusinessLogic.FizzBuzzLogic.java (line 56) a delegate creation expression has been transformed to the creation of an anonymous object that supports the Predicate<Integer> interface (most importantly it has an invoke() method that will run the delegate code). More examples of delegate and event handling can be seen in the Delegates show case.
  • CS2J automatically adds the necessary methods to classes that implement the C# IEnumerable<T> and ICollection<T> interfaces so that they also support the corresponding Java Iterable and Collection classes. An example of this is in FizzBuzz.Core.Range.java where an iterator() has been added. Another example is shown in the Generics show case.
There are many more interesting examples that you can find by studying this example. You can download the code by right clicking on the top most folder. Both are runnable code.
